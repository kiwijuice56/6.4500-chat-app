Chat App Prototype

What/where/who

Note: I am using the term "thread" instead of "chat" internally because it better corresponds to the free entering/leaving aspect, but "chat" is more intuitive for users.

  1. Create thread
    • What: create a new thread with a title, set of tags, and size limit
    • Where: findable by anyone, so using a hardcoded channel such as "designftw-26"
    • Who: anyone can see created threads, so no allowed list is necessary
  2. Join thread
    • What: join an existing thread
    • Where: a private channel that belongs to the thread
    • Who: anyone in thread
  3. Leave thread
    • What: leave an existing thread
    • Where: a private channel that belongs to the thread
    • Who: anyone in thread
  4. Send message
    • What: send a message in a particular thread
    • Where: a private channel that belongs to the thread
    • Who: anyone in thread

AI Usage

Strategy

As recommended by the assignment, I attempted to "vibe code" the spine of the website using Vue and Graffiti. I used Claude instead of Cursor to minimize the friction of needing to install another application. Overall, the results were surprisingly good. I first created my list of who/what/where actions, then fed those actions into the AI along with the assignment instructions, example code from studio, and a high-level specification of the app functionality. I evaluated the code by reviewing it and comparing it with the studio code in addition to testing the website manually. It ran almost perfectly on the first try, but I had to feed it an additional prompt to refine ambiguous behavior that I specified in the first prompt. Although I am wary of using AI for long-term projects that require maintenance, I think it greatly sped up my coding process this time. It might not fare so well with more complex projects. I decided to not use any AI while styling the skeleton in order to familiarize myself with the structure of the code and keep my CSS clean for future additions. I based the design on the Figma prototype.
